Releases

  • 8.3.2 Current, March 2024 Current version for everyday user
  • 8.4.0 Preview, Daily Latest reviewed functional code
  • 7.8.8 Legacy, August 2023 Maintained legacy version for some production systems

New features in GRASS 8.4

Tools (i.svm*, v.fill.holes, notable fixes in r.horizon - ask natrx if they want to be mentioned)
API and Library changes (improvements in grass.script - create location/project without a running session and grass.jupyter - SeriesMap)
GUI (Preferences, History tab, modeler integrated in single window, undockable map display windows, notable fixes in d.* modules when accessed from cli - much faster rendering)
Renaming location to project - community feedback
GRASS evolves, it is not a monolith
Community contributed extensions r.flowaccumulation, r.fusion, r.windfetch, r.maxent.train, r.maxent.predict plotting tools, downloading data tools,
Upcoming stuff

Code quality and security

Code quality and security tools run in CI with every PR/commit. The output is in logs, but also as a suggestion in the PR itself. These suggestions are made by reviewdog.

C and C++

  • Issues reported by GCC and Clang such as dangling-else, discarded-qualifiers, format, … fixed.

Python

  • Code formatted with Black
  • Flake8 enabled, 2/3 fixed
  • Pylint partially enabled

Code security

  • Bandit for vulnerabilities in Python
  • CodeQL for security in C
  • Coverity for security and code quality in C++ - runs once a day
